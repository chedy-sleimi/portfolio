# Product

## Register

brand

## Platform

web

## Users

The one reader every tradeoff optimizes for: an EU recruiter (Spain, Portugal,
Poland) screening a junior QA candidate — usually non-technical, skimming for 10
seconds, arriving from a CV link or LinkedIn. Secondary: a technical interviewer
reading the Neotest case study before a screen; a francophone recruiter reading
`/fr/`.

## Product Purpose

Turn a skim into a conversation. Show precision, prove capability through Neotest,
make reaching out effortless. Success = interview invites.

## Positioning

QA engineer who builds the tools that test — from ISTQB test design to an
LLM-powered automation platform.

## Conversion & proof

- Primary CTA: contact (email). Secondary: download CV (PDF), read the Neotest case study.
- The line a visitor remembers: "QA engineer who builds the tools that test."
- Belief ladder: (1) meticulous — the site itself is flawless; (2) can automate —
  Playwright, API testing, CI/CD; (3) builds beyond running tests — Neotest;
  (4) easy to reach — visible email, plain availability.
- Proof: the Neotest case study (words + pipeline only, see ADR-0004), GitHub
  profile, the site's own repo.

## Brand Personality

Precise and rigorous; quiet confidence. Honesty as strategy: visa status and
availability stated plainly (About/Contact + footer), never hidden, never in the hero.

## Anti-references

- Dev-terminal aesthetic (fake terminals, monospace-everything, hacker green)
- Flashy animation showcase (parallax, particles, intros)
- Template portfolio grammar (card grids, gradient heroes, eyebrow labels)

## Design Principles

1. The site is the test suite — a broken detail contradicts the entire pitch.
2. Show, don't tell — Neotest demonstrated through its pipeline, not adjectives.
3. 10-second skim first — who, what, one-line claim land immediately; depth on `/neotest/`.
4. Quiet confidence — restraint over spectacle; precision carries the personality.

## Accessibility & Inclusion

WCAG AA: ≥4.5:1 body contrast (verified against tokens), keyboard navigation, skip
link, `prefers-reduced-motion` cover, semantic landmarks, correct `lang` per locale.
